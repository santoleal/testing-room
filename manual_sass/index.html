Pasos para instalar SASS

El ambiente de exploración es VSCode

  1. Siempre hacer backup de archivos css en caso de que ya existan (y se haga una migración de estilos a scss).
  2. Crear carpeta/archivo main.scss
  3. Abrir nueva terminal en VSCode.
  4. Iniciar npm init. Aceptar y crear package.json
  5. Crear script "start (nombre opcional): 'sass --watch sass/main.scss:css/style.css' " y editar metadata a criterio. No olvidar guardar cambios.
  6. En el caso anterior, yo coloqué arbitrariamente la ruta para que sean compilados los css. Se debe cambiar de acuerdo a cada proyecto.
  7. Iniciar la compilación con el comando: npm start.
  8. Para finalizar la compilación, desde terminal presionar [ CTRL-C ].

Iniciarse en el uso de SASS

Un ayudamemoria y algunas recomendaciones.

  1. En arhivo principal de SASS -en mi caso, main.scss-, comenzar por importar todos los archivos modulares a usar en SASS.
  2. Estos archivos deben crearse con un guión bajo al comienzo "_". Por ejemplo, "_menu.scss", o "_footer.scss". De eso se trata la modulación, para generar mayor claridad del área que afecta.
  3. Para importar, se hace a través de "@import './nombre_de_archivo'".
  4. Los primeros archivos en ser importados son las variables y luego los mixins.

Sintáxtis: Nesting o anidación

Sutil diferencia con CSS, vinculada estrechamente a la estructura HTML.

  1. El nesting es una forma de ordenar con identaciones.
  2. El anidamiento es cuando dentro de una llave, un selector padre contiene a sus hijos.
  3. Esta es una estructura más precisa.
  4. La identación formal es de UN TAB o cuatro espacios.
  5. Es similar a la estructura de anidación de los @media queries.

Sasscript

Su nombre especial para algo que se instuye...

  1. Es el lenguaje de secuencia de comandos que se usa en SASS.
  2. Se usa para variables, operaciones, funciones...
  3. Esto logra mayor dinamismo y flexibilidad.

Importar

Una de las mejores características de SASS.

  1. Con esto, SASS permite modularizar, separar archivos.
  2. Archivo principal (por convención, "main.scss") sólo tendrá los archivos importados.
  3. Esto aporta limpieza.
  4. Se importa con @import './direccion_de_archivo.scss', antecedido por un guió bajo "_nombre.scss".

Variables (vars)

Son sólo acotadas a propiedades específicas.

  1. Una variable almacena valores, y sirven para ser reutilizadas en otros lugares del código.
  2. Llevan el símbolo $.
  3. Puede ser variable global o variable local.
  4. La variable global está fuera del nesting, posiblemente en un archivo externo (en mi caso, "_variables.scss").
  5. La variable local se describe dentro de los corchetes { } de cada selector donde se quiera usar la variable.

Operaciones

  1. Se pueden usar operaciones básicas.
  2. Están los símbolos clásicos: "+", "-", "/" y "*", los cuales se utilizan después de cada atributo numérico.

Condicionales IF / ELSE

Depende de una condición (heureka!) para ejecutarse

Directiva IF...
  1. Es una forma de control. Se ejecuta si la respuesta es TRUE.
    2. Directiva ELSE...
  2. En el caso que el IF sea FALSE, se puede ejecutar un ELSE...

    Ejemplo:
    @if $tamanio1 > $tamanio2 {
    color: black; } else {
    color: red; }

Condicionales: Bucles (ciclo for, o para)

Porción de código que se repite hasta cumplir una condición.

  1. Sintaxis: @for
  2. Palabras claves: THROUGH / TO.

    @for $var from (inicio) to (final) { ... }. Se incluye el valor final.

    @for $var from (inicio) through (final) { ... }. Se excluye el valor final.
  3. El (inicio) y (final) deben ser números enteros.

Directiva EACH

De cumplirse una pequeña característica se ejecuta una acción

  1. Ideal en estilos repetitivos con pocas variaciones entre ellas.
  2. En el bucle se puden indicar otros parámetros que los numñericos

    Ejemplo: instagram, twitter, ...

    @each $red-social in face, insta, twitter { .#{$red-social} { atributo:propiedad; } }

Directiva MAPS

Permite construir objetos o entidad

  1. Permite agrupar llaves y valores en un objeto.
  2. Se puede complementar con un buclé.
  3. Los objetos en programación son objetos en la vida real, con características concretas.
  4. Permite diseñar un mapa:
    $nombre_de_mapa { llave1: valor1; llave2: valor2; }
    Y luego se somete a un operador.
  5. O sea:
    @each $llave, $valor in $nombre_de_mapa { .llave--#{$llave} { background: $valorX;... } }

Directiva MAps-get()

Función que extrae elementos puntuales de los mapas

  1. Busca los elementos que deseamos.
  2. Ejemplo:
    .claseXX { background: map-get($nombre_de_mapa, llaveX); }

Directiva Extend

Comparte propiedades de un elemento de un selector a otro.

  1. Es para cuando se comparte casi todo, meno algunis detalles.
  2. Se basa en concepto de herencia.
  3. Se llama dentro de las llaves { } con @extend .clase_llamada;
  4. Luego se pueden agregar las propiedades particulares.

Directiva MIXIN e INCLUDE

Se va a retutilizar en todo el sitio

  1. Sintaxis del @mixin:
    @mixin nombre-del-mixin { propiedad: valor; propiedad: valor; ... }
  2. Sintaxis del @include:
    selector { @include nombre-del-mixin; }
  3. Permite pasar parámetros, variables a mixin.
  4. En la misma línea, los mixin aceptan argumentos.
  5. Ejemplo:
    @mixin borde($ancho, $color) { borde: $ancho solid $color; } .argumento1 { @include borde(2px, #ccc);
    }
    .argumento2 { @include borde(4px, #fff); }
